# Product

## Register

product

## Users

เจ้าของเดี่ยว / ฟรีแลนซ์ ที่ใช้แอปนี้ติดตามงานของตัวเองเป็นประจำทุกวัน ไม่ใช่ทีมหรือองค์กร ไม่มีหน้าขาย/การตลาด ผู้ใช้คือคนคนเดียวกับที่ดูแลโค้ด

บริบทการใช้งาน: เปิดดูหลายครั้งต่อวันบนเดสก์ท็อป (และบางครั้งบนมือถือ) เพื่อ
- จัดการงานที่ค้าง/กำลังทำ บน Kanban
- บันทึกสิ่งที่ทำในแต่ละวัน + ชั่วโมงที่ใช้
- ดูสรุปย้อนหลังรายวัน/รายสัปดาห์ในหน้า Reports

Job to be done: "วันนี้ต้องทำอะไร, ทำอะไรไปแล้ว, สัปดาห์นี้เป็นยังไง" ให้ตอบได้ในไม่กี่วินาที โดยไม่ต้องคิดเยอะ

## Product Purpose

เครื่องมือบันทึกและติดตามงานรายวันส่วนตัว (Kanban + Daily Log + Reports) ที่เบา เปิดเร็ว เก็บข้อมูลในเครื่อง (localStorage) ไม่ต้องล็อกอิน ไม่มี backend

ความสำเร็จ = ผู้ใช้เปิดใช้จริงทุกวันโดยไม่รู้สึกฝืน: เพิ่ม/ขยับงานได้ลื่น บันทึกงานได้ไว และเห็นภาพรวมได้ทันทีโดยไม่ต้องตีความ

## Brand Personality

สดใสแต่มีวินัย (energetic, but disciplined). เก็บอัตลักษณ์โทนม่วง-ชมพูและความมีชีวิตชีวาไว้ แต่ใช้สีและ emoji อย่างมีจุดประสงค์ ไม่หว่านจนรก โทนเป็นมิตร ให้กำลังใจ เหมือนผู้ช่วยส่วนตัวที่กระตือรือร้นแต่ไม่จุกจิก

3 คำ: สดใส (bright) · เป็นมิตร (friendly) · คุมโทน (composed)

## Anti-references

- **แอปองค์กรไร้ชีวิต** (Jira/ระบบองค์กรเก่า): เทาทึม แน่น เย็นชา ใช้ยาก — สิ่งที่ต้องหลีกเลี่ยงที่สุด
- **SaaS template โหลๆ ที่ AI เจนเกิน**: gradient จัดจ้านทุกที่, hero metric การ์ดซ้ำๆ icon+heading+text เรียงไม่จบ, eyebrow ตัวเล็กพิมพ์ใหญ่เหนือทุก section
- **รก/แน่นเกินไป**: ยัดข้อมูลทุกอย่างในจอเดียว สีเยอะจนเวียนหัว ไม่มีที่ให้สายตาพัก

## Design Principles

1. **ใช้จริงทุกวัน เป็นตัวตัดสิน** — ความเร็วและความลื่นของงานที่ทำซ้ำบ่อย (เพิ่มงาน, ขยับการ์ด, บันทึกชั่วโมง) สำคัญกว่าความหวือหวา ลด friction ทุกจุดที่ทำได้
2. **อ่านออกก่อนสวย** — ข้อมูล (ชื่องาน, ชั่วโมง, สถานะ) ต้องชัดและ contrast พอเสมอ การตกแต่งห้ามมาบังการอ่าน
3. **สดใสอย่างมีวินัย** — สีและ emoji เป็นเครื่องมือสื่อความหมาย (สถานะ, หมวดหมู่, ความสำคัญ) ไม่ใช่ของประดับ ถ้าไม่สื่อความหมายให้ตัดทิ้ง
4. **หนึ่งหน้าจอ หนึ่งงานหลัก** — แต่ละหน้า (ภาพรวม/Kanban/บันทึก/รายงาน) มีงานหลักชัดเจนหนึ่งอย่าง อย่างอื่นเป็นรอง
5. **มีชีวิตแต่ไม่จุกจิก** — micro-interaction และ feedback ที่ให้กำลังใจได้ แต่ต้องเบา เร็ว และเคารพ reduced-motion

## Accessibility & Inclusion

ไม่มีข้อกำหนดทางการ (ผู้ใช้คือคนเดียว สายตาปกติ) แต่ตั้ง baseline ที่ดีไว้:
- Body text contrast ≥ 4.5:1, ข้อความใหญ่/หนา ≥ 3:1 ทั้งโหมดสว่างและมืด
- รองรับ `prefers-reduced-motion` สำหรับทุก animation
- เป้าแตะบนมือถือพอเหมาะ, โฟกัสคีย์บอร์ดมองเห็นได้
- ไม่พึ่งสีอย่างเดียวในการสื่อสถานะ (มี icon/ข้อความกำกับ)
